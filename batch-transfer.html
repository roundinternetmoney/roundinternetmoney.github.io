Transfers

Execution is driven from scanned wallet balances on the selected chain. The page does not require manual token or amount entry.

Token execution is driven from scanned wallet balances. Default behavior is full-balance handling for all detected tokens on the selected chain.
For selected_known_tokens or all_known_tokens, sequential_calls sends every selected token balance to the recipient, while permit_batch_signer performs sequential ERC-20 approvals to the approval target contract. Rapid submit skips confirmation waits and pushes the next wallet request after the transaction hash is returned.
Scan balances to populate the token transfer list.
Execution applies to scanned token balances only. Amounts default to the full detected balance for each selected token.
